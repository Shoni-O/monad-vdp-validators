## Research Instructions

### For Each Validator:

1. **Visit Website**: Follow the website link to find geographic/infrastructure clues
2. **Check Social Media**: Look for @handles in X/Twitter profiles (linked in metadata)
3. **GitHub Analysis**: Search for infrastructure repos showing compute locations
4. **DNS/Infrastructure**: Use whois/netstat tools to identify hosting providers
5. **Support Pages**: Contact/About pages often reveal headquarters location

### Fill-in Process:

- **Country**: ISO country code (US, DE, JP) or full name (United States, Germany, Japan)
- **City**: City where infrastructure/HQ is based
- **Provider**: Hosting provider (AWS, Hetzner, OVH, Linode, etc.) or datacenter
- **Evidence/Source**: Where you found this info (website, GitHub, whois lookup, etc.)
- **Confidence**: HIGH / MEDIUM / LOW based on verification strength

### High-Confidence Sources (Preference Order):
1. Official about/contact pages explicitly stating location
2. GitHub repos with infrastructure documentation
3. Whois/DNS records for registered domains
4. Multiple corroborating sources (twitter bio + website + github)

### Medium-Confidence Sources:
1. Single explicit mention on website
2. LinkedIn company profiles
3. DNS provider location (not always accurate for CDNs)

### Low-Confidence Sources:
1. Inferred from timezone/language
2. IP geolocation (ISP may not match actual location)
3. Speculative based on validator name
